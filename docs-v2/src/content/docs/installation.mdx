This page covers how to set up a local environment for Solana development on the stable v1 (`anchor-lang`) path. For the v2 alpha, see [v2 installation](/docs/v2/installation).

## Quick installation

A single script installs Rust, the Solana CLI, the Anchor CLI, Surfpool, Node.js, and Yarn in one go.

<Tabs groupId="os" items={["Mac/Linux", "Windows"]}>
<Tab>
Run the installer in your terminal:

```ansi
$ [34mcurl[0m [2m--proto[0m '=https' [2m--tlsv1.2[0m [2m-sSfL[0m https://solana-install.solana.workers.dev | [34mbash[0m
```
</Tab>
<Tab>
<Callout variant="warning" title="WSL required">
  Solana program development on Windows requires [WSL](https://learn.microsoft.com/en-us/windows/wsl/install). All commands below run inside the Ubuntu (Linux) terminal, not PowerShell.
</Callout>

Install WSL from PowerShell with `$ [34mwsl[0m [2m--install[0m{:ansi}`, restart your machine, and create a default Ubuntu user when prompted. Then open the Ubuntu terminal and run:

```ansi
$ [34mcurl[0m [2m--proto[0m '=https' [2m--tlsv1.2[0m [2m-sSfL[0m https://solana-install.solana.workers.dev | [34mbash[0m
```
</Tab>
</Tabs>

After installation, you should see output similar to the following:

```ansi
Installed Versions:
Rust: rustc 1.95.0 (59807616e 2026-04-14)
Solana CLI: solana-cli 3.1.14 (src:ca2c682f; feat:534737035, client:Agave)
Anchor CLI: anchor-cli 0.32.1
Surfpool CLI: surfpool 1.2.0
Node.js: v24.12.0
Yarn: 1.22.22

Installation complete. Please restart your terminal to apply all changes.
```

<Callout>
  If the quick install succeeds, skip ahead to [Solana CLI basics](#solana-cli-basics) and [Anchor CLI basics](#anchor-cli-basics). If it fails, follow the per-tool instructions below.
</Callout>

## Manual installation

The walkthroughs below install each dependency individually.

<Step title="Install Rust">

Solana programs are written in [Rust](https://www.rust-lang.org/). The recommended installer is [rustup](https://www.rust-lang.org/tools/install):

```ansi
$ [34mcurl[0m [2m--proto[0m '=https' [2m--tlsv1.2[0m [2m-sSf[0m https://sh.rustup.rs | [34msh[0m [2m-s[0m -- [2m-y[0m
```

A successful install prints:

```ansi
[1mRust is installed now. Great![0m

To get started you may need to restart your current shell.
This would reload your [1mPATH[0m environment variable to include
Cargo's bin directory ($HOME/.cargo/bin).
```

Reload your `PATH` to pick up the `~/.cargo/bin/` directory:

```ansi
$ . "$HOME/.cargo/env"
```

Verify the install with `$ [34mrustc[0m [2m--version[0m{:ansi}`:

```ansi
$ [34mrustc[0m [2m--version[0m
rustc 1.95.0 (59807616e 2026-04-14)
```

</Step>

<Step title="Install Solana CLI">

The Solana CLI ships the tools required to build and deploy Solana programs. Install with the official script:

```ansi
$ [34msh[0m [2m-c[0m "$(curl [2m-sSfL[0m https://release.anza.xyz/stable/install)"
```

<Callout variant="tip" title="Pinning a version">
  Replace `stable` with a release tag (e.g. `v2.0.3{:bash}`) or one of `beta` / `edge`. All available releases are listed on the [Agave GitHub repo](https://github.com/anza-xyz/agave/releases).
</Callout>

On macOS with `zsh`, append the `PATH` export to `~/.zshrc` so it persists between shell sessions:

```ansi
$ [34mecho[0m 'export PATH="$HOME/.local/share/solana/install/active_release/bin:$PATH"' >> ~/.zshrc
```

Reload the shell with `$ [34msource[0m ~/.zshrc{:ansi}`, then verify with `$ [34msolana[0m [2m--version[0m{:ansi}`:

```ansi
$ [34msolana[0m [2m--version[0m
solana-cli 3.1.14 (src:ca2c682f; feat:534737035, client:Agave)
```

<Callout>
  Agave is the validator client from [Anza](https://www.anza.xyz/), formerly known as the Solana Labs validator client.
</Callout>

To update the Solana CLI later, run `$ [34magave-install[0m update{:ansi}`.

</Step>

<Step title="Install Anchor CLI">

[Anchor](https://www.anchor-lang.com/) is a framework for developing Solana programs that uses Rust macros to reduce boilerplate. The recommended install path is the Anchor Version Manager (AVM), which tracks multiple Anchor versions and simplifies upgrades.

## Solana CLI basics

The following steps cover the most common Solana CLI commands. These work identically on every operating system.

<Steps>
<Step>

### Solana config

Inspect the current config with `$ [34msolana[0m config get{:ansi}`:

```ansi
$ [34msolana[0m config get
[1mConfig File:[0m /Users/test/.config/solana/cli/config.yml
[1mRPC URL:[0m https://api.mainnet-beta.solana.com [3m[0m
[1mWebSocket URL:[0m wss://api.mainnet-beta.solana.com/ [3m(computed)[0m
[1mKeypair Path:[0m /Users/test/.config/solana/id.json [3m[0m
[1mCommitment:[0m confirmed [3m[0m
```

`RPC URL` and `WebSocket URL` identify the cluster the CLI targets. The default is `mainnet-beta`. To switch clusters, use either the long (`--url`) or short (`-u<letter>`) form:

| Cluster         | Long                                   | Short                   |
|-----------------|----------------------------------------|-------------------------|
| mainnet-beta    | `$ [34msolana[0m config set [2m--url[0m mainnet-beta{:ansi}` | `$ [34msolana[0m config set [2m-um[0m{:ansi}` |
| devnet          | `$ [34msolana[0m config set [2m--url[0m devnet{:ansi}`       | `$ [34msolana[0m config set [2m-ud[0m{:ansi}` |
| localhost       | `$ [34msolana[0m config set [2m--url[0m localhost{:ansi}`    | `$ [34msolana[0m config set [2m-ul[0m{:ansi}` |
| testnet         | `$ [34msolana[0m config set [2m--url[0m testnet{:ansi}`      | `$ [34msolana[0m config set [2m-ut[0m{:ansi}` |

`Keypair Path` points at the default wallet used to sign transactions and pay fees, located at `~/.config/solana/id.json`. The next step generates a keypair at that location.

</Step>
<Step>

### Create wallet

The Solana CLI requires a keypair wallet funded with SOL to interact with the network. Generate one at the default keypair path:

```ansi
$ [34msolana-keygen[0m new
Generating a new keypair

For added security, enter a BIP39 passphrase

NOTE! This passphrase improves security of the recovery seed phrase NOT the
keypair file itself, which is stored as insecure plain text

BIP39 Passphrase (empty for none):

Wrote new keypair to /Users/test/.config/solana/id.json
===========================================================================
pubkey: 8dBTPrjnkXyuQK3KDt9wrZBfizEZijmmUQXVHpFbVwGT
===========================================================================
Save this seed phrase and your BIP39 passphrase to recover your new keypair:
cream bleak tortoise ocean nasty game gift forget fancy salon mimic amazing
===========================================================================
```

<Callout variant="note">
  If a wallet already exists at the default path, `$ [34msolana-keygen[0m new{:ansi}` will not overwrite it unless you pass `[2m--force[0m{:ansi}`.
</Callout>

Print the public key of the wallet with `$ [34msolana[0m address{:ansi}`:

```ansi
$ [34msolana[0m address
8dBTPrjnkXyuQK3KDt9wrZBfizEZijmmUQXVHpFbVwGT
```

</Step>
<Step>

### Airdrop SOL

Your wallet needs SOL to pay transaction fees and deployment costs. Point the CLI at devnet:

```ansi {3,4}
$ [34msolana[0m config set [2m-ud[0m
[1mConfig File:[0m /Users/test/.config/solana/cli/config.yml
[1mRPC URL:[0m https://api.devnet.solana.com [3m[0m
[1mWebSocket URL:[0m wss://api.devnet.solana.com/ [3m(computed)[0m
[1mKeypair Path:[0m /Users/test/.config/solana/id.json [3m[0m
[1mCommitment:[0m confirmed [3m[0m
```

Then request an airdrop with `$ [34msolana[0m airdrop 2{:ansi}`.

<Callout variant="important" title="Rate limits">
  The public faucet rate-limits airdrops per IP. If you see this error, use the [Solana Web Faucet](https://faucet.solana.com) instead.

  ```ansi
  $ [34msolana[0m airdrop 2
  Requesting airdrop of 2 SOL
  Error: airdrop request failed. This can happen when the rate limit is reached.
  ```
</Callout>

Check the wallet balance with `$ [34msolana[0m balance{:ansi}`:

```ansi
$ [34msolana[0m balance
2 SOL
```

</Step>
<Step>

### Run local validator

## Anchor CLI basics

The following steps cover the most common Anchor CLI commands. For full reference documentation, see the [Anchor docs](https://www.anchor-lang.com/docs).

<Steps>
<Step>

### Initialize project

Create a new Anchor project with `$ [34manchor[0m init [2m<project-name>[0m{:ansi}`.

This creates a directory named `[2m<project-name>[0m{:ansi}` and scaffolds a modular Rust program. Tests default to the [LiteSVM](https://github.com/LiteSVM/litesvm) crate. To choose a different test template (`mollusk`, `mocha`, `jest`, etc.), pass `[2m--test-template[0m{:ansi}`:

```ansi
$ [34manchor[0m init [2m<project-name>[0m [2m--test-template[0m mollusk
```

<Callout variant="note">
  The default modular layout separates instructions, state, constants, and errors into distinct files. This structure is recommended for any codebase you plan to maintain. See the [project file structure](/docs/quickstart/local#project-file-structure) for details.
</Callout>

Navigate into the new directory with `$ [34mcd[0m [2m<project-name>[0m{:ansi}`.

</Step>

<Step>

### Build program

Compile the program with `$ [34manchor[0m build{:ansi}`. The compiled artifact is written to `target/deploy/`.

</Step>
<Step>

### Deploy program

Deploy the program to the cluster specified in [`Anchor.toml`](https://www.anchor-lang.com/docs/references/anchor-toml) with `$ [34manchor[0m deploy{:ansi}`.

</Step>
<Step>

### Test program

Run the project's test suite with `$ [34manchor[0m test{:ansi}`.

<Callout>
  `$ [34manchor[0m test{:ansi}` builds, deploys, and runs the tests. When `cluster = "Localnet"{:toml}` in `Anchor.toml`, it also starts a local validator, runs the tests against it, and shuts it down afterward.
</Callout>

</Step>
</Steps>

## Shell completions

`anchor` and `avm` can generate completions for `bash`, `fish`, and `zsh`.

<Tabs groupId="shell-completions" items={["bash", "fish", "zsh"]}>
  <Tab>
    ```ansi
    $ [34mmkdir[0m [2m-p[0m $HOME/.local/share/bash-completion/completions
    $ [34manchor[0m completions bash > $HOME/.local/share/bash-completion/completions/anchor
    $ [34mavm[0m completions bash > $HOME/.local/share/bash-completion/completions/avm
    $ [34mexec[0m bash
    ```
  </Tab>
  <Tab>
    ```ansi
    $ [34mmkdir[0m [2m-p[0m $HOME/.config/fish/completions
    $ [34manchor[0m completions fish > $HOME/.config/fish/completions/anchor.fish
    $ [34mavm[0m completions fish > $HOME/.config/fish/completions/avm.fish
    $ [34msource[0m $HOME/.config/fish/config.fish
    ```
  </Tab>
  <Tab>
    First ensure the following is in your `~/.zshrc` (skip this if using `oh-my-zsh`):

    ```ansi
    autoload [2m-U[0m compinit
    compinit [2m-i[0m
    ```

    Then generate the completions:

    ```ansi
    $ [34manchor[0m completions zsh | [34msudo[0m tee /usr/local/share/zsh/site-functions/_anchor
    $ [34mavm[0m completions zsh | [34msudo[0m tee /usr/local/share/zsh/site-functions/_avm
    $ [34mexec[0m zsh
    ```
  </Tab>
</Tabs>
