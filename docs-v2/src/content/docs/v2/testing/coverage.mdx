---
title: Coverage
description: Generate LCOV coverage from SBF register traces.
---

`$ [34manchor[0m coverage{:ansi}` maps executed SBF program counters back to Rust source lines and emits LCOV.

```ansi showLineNumbers=false
$ [34manchor[0m coverage
```

By default, output lands at:

```text showLineNumbers=false
target/coverage/sbf.lcov
```

## Options

Useful flags:

| Flag | Effect |
| --- | --- |
| `[2m--skip-run[0m{:ansi}` | Reuse existing traces. |
| `[2m--skip-build[0m{:ansi}` | Skip `$ [34mcargo[0m build-sbf{:ansi}` when deploy artifacts are already fresh. |
| `[2m--output[0m [2m<path>[0m{:ansi}` | Write LCOV somewhere other than `target/coverage/sbf.lcov`. |
| `[2m--trace-dir[0m [2m<dir>[0m{:ansi}` | Read or write traces from a custom directory. |

Coverage uses the same underlying trace data as profiling and debugger, so the same test-harness requirement applies. Tests should call `anchor_v2_testing::svm(){:rs}`.

## DWARF

The command builds programs with DWARF debug info so `addr2line` can resolve PCs to source. If source lines are missing, make sure the deploy `.so` exists and was built by the same source tree that produced the traces.
