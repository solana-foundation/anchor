---

v2 has not been published to `crates.io` yet, so the recommended install path is to build the CLI directly from the [`anchor-next`](https://github.com/solana-foundation/anchor/tree/anchor-next) branch with cargo:

```ansi
$ [34mcargo[0m install [2m--git[0m https://github.com/solana-foundation/anchor.git [2m--branch[0m anchor-next anchor-cli [2m--force[0m
```

<Callout variant="warning" title="Overwrites the avm-managed anchor binary">
  `$ [34mcargo[0m install{:ansi}` overwrites whatever `anchor` binary is on your `PATH`, including one managed by [`avm`](/docs/installation/#anchor-version-manager-avm). To keep v1 alongside v2, run the v2 binary directly from a source checkout (see below) and leave AVM in charge of the `PATH` entry.
</Callout>

## macOS LTO workaround

The default release profile uses link-time optimization, and on macOS the linker may fail mid-build with an error of the form:

```text
ld: could not parse bitcode object file ... Unknown attribute kind
```

If that happens, disable LTO for the duration of the install:

```ansi
$ [34mCARGO_PROFILE_RELEASE_LTO[0m=off [34mcargo[0m install [2m--git[0m https://github.com/solana-foundation/anchor.git [2m--branch[0m anchor-next anchor-cli [2m--force[0m
```

## Building from a local checkout

Building from a local checkout is useful when you want to tweak v2 internals or run `$ [34manchor[0m debugger{:ansi}` against the bench programs that ship in the repo:

```ansi
$ [34mgit[0m clone https://github.com/solana-foundation/anchor.git
$ [34mcd[0m anchor && [34mgit[0m checkout anchor-next
$ [34mcargo[0m install [2m--path[0m cli [2m--force[0m
```

Prepend `[34mCARGO_PROFILE_RELEASE_LTO[0m=off{:ansi}` to the install command on macOS for the same LTO reason described above.

After the install completes, either of the following commands resolves the freshly-built v2 binary:

```ansi
$ [34manchor[0m [2m--version[0m
$ [34mwhich[0m anchor
```

## Depending on `anchor-lang-v2` from a program

To pull `anchor-lang-v2` into a program, add a git dependency in `Cargo.toml`:

```toml title="Cargo.toml"
[dependencies]
anchor-lang-v2 = { git = "https://github.com/solana-foundation/anchor.git", branch = "anchor-next" }
```

Programs that integrate with SPL tokens should also pull in `anchor-spl-v2`:

```toml title="Cargo.toml"
[dependencies]
anchor-spl-v2 = { git = "https://github.com/solana-foundation/anchor.git", branch = "anchor-next" }
```

For host-side tests that record register traces (which feed flamegraphs and the debugger), depend on `anchor-v2-testing` with the `profile` feature:

```toml title="Cargo.toml"
[dev-dependencies]
anchor-v2-testing = { git = "https://github.com/solana-foundation/anchor.git", branch = "anchor-next", features = ["profile"] }
```

See the [Feature flags](/docs/v2/feature-flags/) page for the full set of opt-in toggles and how they interact.
