`anchor-lang-v2` is the next-generation Anchor runtime, built on [pinocchio](https://github.com/anza-xyz/pinocchio) and `#![no_std]{:rs}` by default. v2 is a drop-in replacement for v1 that produces up to **94% smaller binaries** and **3 to 6 times fewer CU per instruction**.

<Callout variant="warning" title="Alpha">
  v2 is in alpha. It has not been audited, is not on `crates.io`, and APIs may break between commits. Depend on it via git on the [`anchor-next`](https://github.com/solana-foundation/anchor/tree/anchor-next) branch. The v1 (`lang/`) code in the same repository remains the stable, published path.
</Callout>

## Why v2

1. v2 swaps v1's `solana_program` foundation for pinocchio and drops `std` entirely, replacing the heavy macro generation with trait dispatch. The bench numbers below quantify what that buys you.

2. Most v1 programs port with the renames in [Migrating from v1](/docs/v2/migration/). Larger programs that lean on v1-shape logging can opt into a `compat` feature that restores helpers like `debug!{:rs}`.

3. Most of v2's logic lives behind traits rather than inside the macro, so the core derive shrinks from ~11,400 LoC to ~3,700 and third-party crates can ship new account primitives and constraint markers without forking. See [Extensibility](/docs/v2/extensibility/).

4. Safer APIs steer users away from common v1 footguns. Whole classes of runtime bugs become compile errors, and fuzzing, static analysis, and formal verification are first-class concerns rather than afterthoughts.

5. `$ [34manchor[0m debugger{:ansi}` rounds out the toolchain. It ships a TUI that steps through SBF instructions per test, built on the same register traces that `$ [34manchor[0m test [2m--profile[0m{:ansi}` consumes for flamegraphs.

## Bench numbers

Worked programs live under [`bench/programs/`](https://github.com/solana-foundation/anchor/tree/anchor-next/bench/programs), paired with v1 for head-to-head comparison.

| Program | Description | Binary | CU range | Bin ↓ | CU ↓ |
| --- | --- | --- | --- | --- | --- |
| `helloworld` | Single-instruction counter | 6.9 KB | 1,383 | 18× | 4.2× |
| `prop-amm` | Oracle feed with asm fast-path | 9.2 KB | 26-1,383 | 15× | 3.1-50× |
| `vault` | Single-depositor SOL vault | 5.9 KB | 403-1,910 | 18× | 3.0-6.1× |
| `nested` | Shared-validation via `Nested<T>{:rs}` | 13 KB | 476-2,748 | 12× | 7.2-10× |
| `multisig` | Four-instruction SOL multisig | 31 KB | 477-2,363 | 5.3× | 3.0-9.1× |
